## 已修正的問題

### P1：熱量公式需要 age / sex / activity，但資料模型缺欄位

工程計畫選擇 Mifflin-St Jeor，並要求目標設定加入年齡、性別、活動量。但原本 `Goal` model 只有身高、體重、目標體重和方向，實作時會造成公式沒有資料可算。

已修正：
- `Goal` 新增 `age`
- `Goal` 新增 `sex`
- `Goal` 新增 `activityLevel`
- `Goal` 新增 `bmr`
- `Goal` 新增 `tdee`

### P2：Vite `public/` 目錄位置容易被誤讀

工程計畫原本目錄草圖把 `public/` 放在 `src/` 底下。現在 source code 會放在 repo root 的 `health-track/` 子目錄，因此 Vite 專案根目錄是 `health-track/`，PWA manifest / icons 應放在 `health-track/public/`。

已修正：
- 目錄草圖改為 `health-track/public/`
- workstream 表格改為 `health-track/public`

### P2：office-hours 文件仍把平台選型列為未決

後續工程計畫已經選擇 mobile-first PWA-lite 與 local-first，但 office-hours 文件仍寫「mobile-first web、native mobile、desktop web」和「local-only 或帳號同步」是未決問題。

## 仍需注意的問題

### P2：PWA 更新策略還不夠具體

決策更新：使用 `autoUpdate`。

原因：
- 目前 app local-first，沒有多人協作狀態。
- MVP 使用者最怕的是舊版 cache 造成看不到修正。
- 如果之後資料 migration 變複雜，再改成 prompt + changelog。

### P2：PWA icon 規格還可以更明確

決策更新：icon 先用簡單健康追蹤符號，不做品牌大工程。

建議：
- 視覺方向：白底、深綠 / teal 線條、抽象進度環 + 小折線。
- 不用食物 emoji。
- 不用醫療十字，避免看起來像醫療 app。
- 不用複雜插畫，因為小尺寸 icon 會糊。
- 至少補以下檔案：
- 192x192 PNG
- 512x512 PNG
- maskable 512x512 PNG
- `theme_color`
- `background_color`
- `display: standalone`

### P2：資料匯出 / 匯入被延後是合理的，但風險要記住

local-first MVP 是對的，但健康紀錄是個人資料。如果使用者清除瀏覽器資料，IndexedDB 會消失。現在 `/settings` 會說明限制，但沒有匯出能力。

決策更新：MVP 先不做 export/import，後續再談。

保留提醒：
- 健康紀錄是個人資料。
- `/settings` 必須清楚說明資料儲存在本機，清除瀏覽器資料可能會移除紀錄。
- 之後如果要補資料可攜性，JSON 匯出 / 匯入仍比雲端同步便宜很多。

### P2：mockup 缺失不是阻擋，但會提高實作跑偏機率

`/plan-design-review` 嘗試產生 mockup，但因為沒有 OpenAI API key，所以只能產生文字版 UI/UX plan。

建議：
- 可以先照 `docs/ui-ux-plan-design-review-20260408-074037.md` 實作。
- 第一個可跑版本完成後，務必跑 `/design-review` 做視覺 QA。

### P3：文件語言策略

決策更新：不用全部翻成中文，好溝通即可。

建議：
- 使用中文描述產品、流程、決策。
- `PWA`、`IndexedDB`、`service worker`、`React Hook Form`、route path、TypeScript 欄位名保留英文。
- 不為了翻譯犧牲實作清楚度。

### P2：source code 位置

決策更新：source code 放在專案根目錄的 `health-track/` 子目錄底下。

建議：
- app scaffold 放在 `health-track/`
